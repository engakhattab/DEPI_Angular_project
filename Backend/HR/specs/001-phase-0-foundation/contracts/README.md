# API Contracts: Phase 0 Foundation

No changes are made to the API contracts during this phase. 

All existing endpoints must retain their current routes, HTTP methods, request bodies, and response shapes to ensure 100% backward compatibility. The goal of Phase 0 is purely structural.

Any modifications to API behavior (such as adding global exception formats or pagination) are strictly reserved for subsequent phases (e.g., Phase 1 of the master plan).
